## Erste Schritte

Um die BusinessGPT API zu nutzen, benötigen Sie:

1. Ein BusinessGPT-Konto mit aktiviertem API-Zugriff
2. Einen API-Schlüssel zur Authentifizierung
3. Verständnis der REST-API-Prinzipien
4. Kenntnis Ihrer spezifischen Integrationsanforderungen

Unsere API folgt RESTful-Prinzipien und verwendet Standard-HTTP-Methoden. Alle Anfragen und Antworten sind im JSON-Format.

## Authentifizierung

Alle API-Anfragen erfordern eine Authentifizierung mit API-Schlüsseln. Fügen Sie Ihren API-Schlüssel im Authorization-Header Ihrer Anfragen ein:

```bash
Authorization: Bearer YOUR_API_KEY
```

<Note>
  Halten Sie Ihre API-Schlüssel sicher und legen Sie sie niemals im Client-seitigen Code offen. Verwenden Sie Umgebungsvariablen oder sichere Tresore, um Ihre Schlüssel zu speichern.
</Note>

## Ratenbegrenzungen

Die API-Nutzung unterliegt Ratenbegrenzungen basierend auf Ihrem Abonnementplan:

| Plan | Ratenbegrenzung | Burst-Limit |
| ---- | ---------- | ----------- |
| Standard | 60 Anfragen/Minute | 100 Anfragen/Minute |
| Professional | 300 Anfragen/Minute | 500 Anfragen/Minute |
| Enterprise | Individuell | Individuell |

Wenn Sie Ihr Ratenlimit überschreiten, gibt die API eine 429 Too Many Requests-Antwort zurück.

## Fehlerbehandlung

Die API verwendet Standard-HTTP-Statuscodes, um Erfolg oder Misserfolg anzuzeigen:

- 2xx: Erfolg
- 4xx: Client-Fehler (ungültige Anfrage)
- 5xx: Server-Fehler

Fehlerantworten enthalten einen JSON-Body mit Details:

```json
{
  "error": {
    "code": "invalid_parameter",
    "message": "Der Parameter 'limit' muss eine positive Ganzzahl sein",
    "details": {
      "parameter": "limit",
      "value": "-5"
    }
  }
}
```

## API-Versionierung

Die API ist versioniert, um die Kompatibilität zu gewährleisten, wenn wir neue Funktionen hinzufügen. Geben Sie die API-Version in der URL an:

```
https://api.businessgpt.eraneos.com/v1/chat/completions
```

Wir gewährleisten Abwärtskompatibilität innerhalb von Hauptversionen.
